# AGENTS.md

## 1. 语言规范
1. 沟通、注释、文档统一简体中文
2. 报错堆栈、API字段、异常类、技术术语保留英文，方便检索
3. 业务长注释中文，短行代码注释可中英混用

## 2. 技术栈约束
1. 前端统一基于 Vue 3 + Composition API 开发，优先使用 SFC 与 `<script setup lang="ts">` 风格，非必要不回退 Options API
2. 类型系统统一使用 TypeScript，禁止无理由新增 `any`；公共类型优先沉淀到 `src/types/`，局部类型可就近定义
3. 状态管理统一使用 Pinia；跨组件共享状态放 `src/stores/`，可复用状态逻辑放 `src/composables/`，避免把状态编排堆进页面组件
4. 目录职责必须清晰：
   - `src/components/` 仅承载界面展示、交互编排与组件内聚逻辑
   - `src/composables/` 放可复用组合式逻辑
   - `src/stores/` 放全局状态、业务编排、跨组件协同
   - `src/utils/` 放纯函数、解析/转换/渲染辅助逻辑，避免耦合 UI 状态
   - `src/plugins/` 放 Milkdown / CodeMirror 扩展插件，禁止把编辑器插件逻辑散落到页面层
5. Markdown、HTML、数学公式、Mermaid 等渲染能力优先复用现有 `utils/` 与 `plugins/` 实现，不重复造轮子，不绕过既有渲染链路
6. uTools 平台桥接统一收口到 `public/preload.js`，前端通过 `window.markflow` 访问宿主能力；禁止在 Vue 代码中直接扩散 Node/utools 专属调用
7. 构建与测试保持现有工具链：Vite 负责开发与打包，Vitest 负责测试；非必要不引入平行构建工具、状态库或测试框架
8. 依赖变更优先复用现有生态（Vue、Pinia、Milkdown、CodeMirror、marked、Vitest），新增依赖前先评估是否已有能力可覆盖

## 3. 代码 & Git 提交规范
### 代码
1. 遵循技术栈标准命名规范
2. 业务文件≤800行，工具类≤500行，超长自动提供拆分方案
3. 无冗余导入、无效判断，输出格式化完整代码

### Git Commit
格式：`[类型] 简述 | 详细变更`
类型：feat/fix/refactor/docs/test/perf/style
示例：`[fix] 金额精度丢失 | 修正Decimal运算，补充边界单测`

## 4. 安全红线（零容忍）
1. 禁止硬编码密码、AK/SK、私钥、Token等敏感凭证
2. 严禁提交.env、密钥、证书文件至版本库
3. 敏感配置统一使用环境变量/密钥管理中心，出现明文密钥直接拦截

## 5. 执行 & TDD 测试规范
1. 严格TDD：先写测试用例，再实现业务代码
2. 生成配套可运行单元测试，覆盖正常、异常、边界场景
3. 报错完整输出原始堆栈，不屏蔽、不伪造成功结果
4. 执行失败附带复现步骤与修复方案
5. 例外规则：纯文档、注释、README 类修改可不强制执行 TDD 与测试；但交付结果中必须明确说明“本次仅文档/注释变更，未运行测试”

## 6. 用户交互规范（效率优先）
1. 少铺垫、无客套，核心内容前置
2. 需求清晰不重复确认；仅模糊/有安全风险时提问
3. 优先提供可直接复制运行代码、命令、脚本
4. 长内容分段分层，结构清晰便于快速查阅

## 优先级
安全红线 > 代码规范 > TDD测试 > 语言规范 > 交互习惯
