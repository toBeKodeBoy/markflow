# MarkFlow v0.0 — MVP 原型

> 基于 [开发计划](./开发计划.md) | 状态：✅ 已完成 | 更新日期：2026-07-04

---

## 1. 版本定位

**v0.0** 是 MarkFlow 的 **最小可用原型（MVP）** 阶段：验证「uTools 插件 + 本地 Markdown 编辑 + 自动保存」这一核心闭环是否可行，不追求完整的产品功能集。

| 维度 | 说明 |
|------|------|
| 目标用户 | 内部验证 / 早期试用 |
| 交付形态 | 可加载的 uTools 插件，能写、能存、能切换笔记 |
| 非目标 | 四视图完备、文件夹、导出、测试体系、富媒体 |

---

## 2. 功能范围

### 2.1 纳入范围（Must Have）

| 模块 | 说明 | 关键实现 |
|------|------|----------|
| 笔记 CRUD | 新建、打开、编辑、删除、自动提取标题 | `src/stores/note.ts` |
| 内容持久化 | uTools `dbStorage` / 开发环境 `localStorage` 双通道 | `src/composables/useStorage.ts` |
| 基础编辑 | 至少一种可输入的 Markdown 编辑方式 | CodeMirror 或 Milkdown 择一落地 |
| 实时保存 | 编辑防抖后写入存储 | `note.ts` + Store |
| uTools 桥接 | preload 暴露读写 API | `public/preload.js` |

### 2.2 明确不包含

- 文件夹分组、标签、排序
- 分屏 / 专注等多视图
- 导入/导出文件、PDF、图片
- 目录导航、滚动同步、大文件策略
- 明暗主题、设置面板
- 自动化测试与 CI

---

## 3. 验收标准

- [x] 在 uTools 中可通过关键词唤起插件
- [x] 可创建多篇笔记并在侧边栏切换
- [x] 编辑内容关闭插件后再次打开，内容不丢失
- [x] 开发环境（浏览器 + `npm run dev`）可正常调试，存储自动回退

---

## 4. 与后续版本关系

```
v0.0 MVP ──→ v1.0 核心版 ──→ v1.1 组织检索 ──→ v1.2 富媒体导出 ──→ v2.0 平台化
  闭环验证      产品化交付        找得到/排得顺       图片/PDF/公式        同步/扩展
```

v0.0 完成后，产品进入 **v1.0** 阶段：补齐四视图、文件夹、导入导出、主题、TOC、测试等「可对外发布」的能力。详见 [v1.0.md](./v1.0.md)。
