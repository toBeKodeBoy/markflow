# MarkFlow v2.0 — 平台化演进

> 基于 [开发计划](./开发计划.md) §5 | 状态：🔮 远期规划 | 更新日期：2026-07-04

---

## 1. 版本定位

**v2.0** 将 MarkFlow 从「单机 Markdown 笔记工具」演进为 **可扩展的写作平台**：支持跨设备同步、多笔记并行编辑，以及可插拔的渲染/导出扩展。

| 维度 | 说明 |
|------|------|
| 前置条件 | v1.2 稳定运行 ≥ 1 个月；用户反馈收集完成；**架构评审通过** |
| 非目标 | 替代 Obsidian / Notion 级别的知识图谱与协作 |
| 核心诉求 | 在 uTools 生态内补齐「多设备」「多任务」「可扩展」三块能力 |

---

## 2. 路线图位置

```
v1.0 ✅ ──→ v1.1 📋 ──→ v1.2 📋 ──→ v2.0 🔮
 核心功能     组织与检索     富媒体与导出     协作与扩展
```

v1.0 / v1.0.1 已完成核心与部分 v1.1、v1.2 能力；**v2.0 不启动开发**，直至 v1.2 交付并稳定。

## 3. 功能规划

### 3.1 云同步（P0 for v2.0）

**目标：** 笔记与图片资产可在多台设备间保持一致，用户掌控数据存放位置。

| 方案 | 优点 | 缺点 | 建议优先级 |
|------|------|------|------------|
| **WebDAV**（坚果云 / Nextcloud 等） | 用户自控、实现相对简单 | 需用户自行配置 | ⭐ 首选 MVP |
| uTools 账号同步 | 零配置 | 依赖平台能力，不可控 | 评估后决定 |
| 自建后端 | 完全可控 | 运维与合规成本高 | 远期备选 |

**建议路径：** WebDAV 同步 MVP → 评估 uTools 官方能力 → 再考虑自建。

#### 任务概要

| # | 任务 | 估时 | 说明 |
|---|------|------|------|
| 3.1.1 | 同步协议设计（笔记 JSON + asset blob + 冲突策略） | 1–2w | Last-write-wins 或版本向量 |
| 3.1.2 | WebDAV 客户端（上传/下载/增量） | 2w | `preload.js` 或 Node 侧 |
| 3.1.3 | 设置页：服务器地址、凭据、同步目录 | 1w | |
| 3.1.4 | 手动同步 + 启动时自动同步（可选） | 1w | |
| 3.1.5 | 冲突 UI：保留本地 / 保留远端 / 合并 | 1w | |

**验收标准：**
- 两台设备间笔记与图片可完整同步
- 离线编辑后再次联网可检测冲突并提示
- 同步过程不阻塞编辑（后台任务）

---

### 3.2 多 Tab 编辑（P1）

**目标：** 同时打开多篇笔记，减少频繁切换带来的上下文丢失。

**现状：** 全局单一 `currentNote`，切换即卸载编辑器内容。

#### 架构改造要点

```typescript
// 目标模型（示意）
interface NoteTab {
  id: string
  noteId: string
  dirty: boolean
}

// Store 改造
openTabs: NoteTab[]
activeTabId: string
```

| # | 任务 | 说明 |
|---|------|------|
| 3.2.1 | `currentNote` → `openTabs` + `activeTabId` | Pinia 重构 |
| 3.2.2 | Tab 栏 UI（新建 / 关闭 / 切换） | `Toolbar` 或独立 `TabBar.vue` |
| 3.2.3 | 未保存提示（切换 Tab / 关闭时） | dirty 标记 |
| 3.2.4 | 内存管理：>5 个 Tab 时卸载非活跃编辑器实例 | 性能 |

**验收标准：**
- 至少 3 个 Tab 同时打开，切换无内容串扰
- 关闭 Tab 前若有未保存修改，弹出确认

---

### 3.3 插件扩展（P2，v2.0 后期）

**目标：** 参考 Obsidian 社区插件的简化版，允许第三方扩展渲染与导出能力。

| 扩展点 | 示例 |
|--------|------|
| 渲染器插件 | Mermaid、PlantUML、自定义 admonition |
| 导出插件 | 自定义 PDF 模板、Word |
| 快捷键插件 | 用户自定义组合键 |

| # | 任务 | 风险 |
|---|------|------|
| 3.3.1 | 定义插件 API（生命周期、钩子） | 高 — 需稳定 ABI |
| 3.3.2 | 插件加载器（本地 JS 模块） | 安全沙箱 |
| 3.3.3 | 官方示例插件 1–2 个 | 文档与 DX |

> **风险说明：** 插件体系改动面大，建议在 v2.0 中后期单独迭代，不阻塞云同步与多 Tab。

## 4. 依赖 v1.x 的前置能力

| v1.x 能力 | 对 v2.0 的意义 | v1.x 状态 |
|-----------|----------------|-----------|
| 数据备份/恢复（ZIP 或 JSON 包） | 同步冲突回滚、迁移 | 📋 v1.1 |
| 图片独立存储（`markflow-asset://`） | 同步需传输 blob | ✅ v1.0.1 |
| 全文搜索 + 标签 | 多 Tab 下快速定位笔记 | 🔄 / 📋 |
| PDF / 富媒体稳定 | 插件扩展的基线渲染 | ✅ PDF；📋 公式 |

---

## 5. 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| uTools API 限制 | 云同步、多窗口能力受限 | 提前与平台能力对齐；WebDAV 走 preload |
| 同步冲突复杂 | 数据丢失或重复 | MVP 用 LWW + 冲突副本文件 |
| 多 Tab 内存 | 大笔记 + 多 Milkdown 实例 OOM | Tab 上限 + 惰性销毁 |
| 插件安全 | 恶意插件读写本地数据 | 签名 / 白名单 / 权限声明 |

## 6. 预估工期

| 阶段 | 范围 | 工期 |
|------|------|------|
| v2.0-alpha | WebDAV 单向备份 + 恢复 | 3–4 周 |
| v2.0-beta | 双向同步 + 冲突 UI | 4–6 周 |
| v2.0-rc | 多 Tab 编辑 | 3–4 周 |
| v2.0 | 插件 API（可选） | 6–8 周+ |

**合计：** 约 **3–6 个月**（含架构评审、联调、uTools 真机验证），具体取决于 v1.2 完成时间与人力。

---

## 7. 不在 v2.0 范围

以下能力明确 **不纳入 v2.0**，避免范围膨胀：

- 实时协作 / 多人编辑
- 移动端独立 App
- 内置 AI 写作助手
- 双链笔记 / 知识图谱
- 替换 uTools 运行时为独立桌面应用
