**1. 标签规范化（`src/utils/tagNormalize.ts`，新）**

```typescript
export const TAG_MAX_LENGTH = 20

/** trim → 去重（大小写不敏感保留首次写法）→ 过滤空串 → 截断/拒绝超长 */
export function normalizeTags(input: string[]): { tags: string[]; rejected?: string } {
  const seen = new Set<string>()
  const tags: string[] = []
  for (const raw of input) {
    const t = raw.trim()
    if (!t) continue
    if (t.length > TAG_MAX_LENGTH) return { tags: [], rejected: t }
    const key = t.toLowerCase()
    if (seen.has(key)) continue
    seen.add(key)
    tags.push(t)
  }
  return { tags }
}

export function normalizeTagInput(raw: string): string | null {
  const t = raw.trim()
  if (!t || t.length > TAG_MAX_LENGTH) return null
  return t
}
```

**2. Store 层 API（`src/stores/note.ts`）**

在现有 `setNoteTags` 基础上扩展细粒度方法，内部统一走 `normalizeTags`：

```typescript
function addTag(id: string, tag: string): boolean {
  const normalized = normalizeTagInput(tag)
  if (!normalized) return false
  const note = storage.getNote(id)
  if (!note) return false
  const { tags } = normalizeTags([...(note.tags ?? []), normalized])
  setNoteTags(id, tags) // 复用已有持久化逻辑
  return true
}

function removeTag(id: string, tag: string): void {
  const note = storage.getNote(id)
  if (!note) return
  const key = tag.toLowerCase()
  setNoteTags(id, (note.tags ?? []).filter((t) => t.toLowerCase() !== key))
}
```

**1. 设计决策：排序作用域**

| 视图 | 排序键 | 说明 |
|------|--------|------|
| 文件夹内 | 同 `folderId` 兄弟节点 | `sortOrder` 仅在同级有效 |
| 「全部笔记」根级 | `folderId === undefined` 的笔记 | 与文件夹内独立编号 |
| 搜索模式 | 命中结果按 `compareNotes` | 不改变底层 `sortOrder` |

跨文件夹移动时（已有 `moveNote`）：重置 `sortOrder = undefined`，再 `bumpNoteSortOrder` 排到目标文件夹末尾。

**2. 排序比较函数（修正 `noteSort.ts`）**

与产品规则对齐：pinned 组内按 `updatedAt` 倒序，非 pinned 按 `sortOrder` 升序（未设置则 `updatedAt` 倒序）。

```typescript
export function compareNotes(a: NoteListItem, b: NoteListItem): number {
  const pinA = a.pinned ? 1 : 0
  const pinB = b.pinned ? 1 : 0
  if (pinA !== pinB) return pinB - pinA

  // pinned 组内：updatedAt 倒序，忽略 sortOrder
  if (pinA === 1) return b.updatedAt - a.updatedAt

  const hasOrderA = a.sortOrder != null
  const hasOrderB = b.sortOrder != null
  if (hasOrderA && hasOrderB && a.sortOrder !== b.sortOrder) {
    return a.sortOrder! - b.sortOrder!
  }
  if (hasOrderA !== hasOrderB) return hasOrderA ? -1 : 1 // 有 sortOrder 的靠前

  return b.updatedAt - a.updatedAt
}
```

```javascript
saveBackupFile: function (jsonString, defaultName) {
  var path = utools.showSaveDialog({
    title: '导出 MarkFlow 备份',
    defaultPath: defaultName,
    filters: [{ name: 'MarkFlow Backup', extensions: ['json'] }]
  });
  if (path) {
    require('fs').writeFileSync(path, jsonString, 'utf-8');
    return { ok: true, path: path };
  }
  return { ok: false, reason: 'cancel' };
},

openBackupFile: function () {
  var paths = utools.showOpenDialog({
    title: '从备份恢复',
    filters: [{ name: 'MarkFlow Backup', extensions: ['json'] }],
    properties: ['openFile']
  });
  if (paths && paths.length) {
    return require('fs').readFileSync(paths[0], 'utf-8');
  }
  return null;
}
```

```
  → applyBackup：folders → notes → merge settings → assets 逐条 saveAsset
  → loadNoteList() + rebuildSearchIndex
  → openNote(first) 或清空 currentNote
```

冲突策略（v1.1 固定）：**全量覆盖**（与现有 confirm 文案一致）；不做合并/增量。v2.0 可扩展「导入为副本（新 ID）」。

**5. 存储用量估算（`src/utils/storageStats.ts`，新）**

```typescript
export interface StorageStats {
  noteCount: number
  folderCount: number
  assetCount: number
  estimatedBytes: number // JSON.stringify 长度之和 + asset.data base64 长度
  estimatedLabel: string // formatBytes → "1.2 MB"
}

export async function estimateStorageUsage(
  storage: BackupStorageReader,
  assets: { getIndex; getAsset }
): Promise<StorageStats>
```

展示位置：`SettingsModal.vue`「数据管理」区顶部只读一行：`共 N 篇笔记 · M 张图片 · 约 X MB`。浏览器开发环境同样可用（localStorage + IndexedDB 估算）。

**6. 批量清理（1.4.4）**

在设置页增加「清空全部数据」按钮，复用已有 `store.clearAllLibraryData()`（```446:459:src/stores/note.ts```），二次 confirm 后执行；与恢复前清库逻辑一致。
