## 1. 产品概述

### 1.1 产品定位

**MarkFlow** 是一款基于 uTools 平台的本地 Markdown 编辑器插件，随叫随到，专注写作体验。面向需要高频记录、写作的技术人员与知识工作者。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| 快捷 | uTools 一键唤起，无需切换窗口，即输即存 |
| 本地 | 所有数据存储在本地，无需网络，隐私安全 |
| 双模式 | 所见即所得（Milkdown）+ 源码编辑（CodeMirror）双引擎 |
| 四视图 | 预览、分屏、源码、专注，适配不同写作场景 |
| 轻量 | 插件化形态，随 uTools 启动，不额外占用系统资源 |

### 1.3 用户画像

| 用户类型 | 典型场景 | 核心需求 |
|----------|----------|----------|
| 开发者 | 写技术文档、API 笔记、代码片段 | 源码编辑、代码高亮、GFM 语法 |
| 知识工作者 | 日常记录、会议纪要、知识整理 | 快捷记录、文件夹组织、全文搜索 |
| 写作者 | 长文写作、博客草稿 | 专注模式、所见即所得、导出 PDF |

## 2. 功能规格

### 2.1 功能全景图

```
MarkFlow
├── 笔记管理
│   ├── 笔记 CRUD
│   │   ├── 新建笔记（自动从标题提取名称）
│   │   ├── 重命名笔记
│   │   ├── 删除笔记
│   │   └── 切换笔记
│   ├── 文件夹管理
│   │   ├── 新建 / 重命名 / 删除文件夹
│   │   └── 按文件夹过滤笔记
│   └── 搜索
│       └── 按标题 + 正文实时过滤（标签待 v1.1）
│
├── 编辑器
│   ├── 四视图：预览 / 分屏 / 源码 / 专注
│   ├── 源码工具栏（加粗/斜体/下划线/标题/列表/代码块等）
│   ├── 行内代码、反引号自动闭合
│   ├── 代码块复制按钮、语言标签切换
│   ├── 下划线（<u>）、==高亮== 语法
│   └── 图片粘贴 / 拖拽（markflow-asset:// 独立存储）
│
├── 预览
│   ├── 实时渲染（防抖节流）
│   ├── GFM + 代码高亮
│   ├── 滚动同步（分屏）
│   ├── 复制 HTML
│   └── 图片缩放、双击全屏预览
│
├── 目录导航
│   ├── 自动解析标题、虚拟列表、点击跳转
│
├── 文件与导出
│   ├── 导入 / 导出 .md
│   └── 导出 PDF（uTools printToPDF / 浏览器打印回退）
│
├── 设置（⚙）
│   ├── 主题：跟随系统 / 浅色 / 深色
│   ├── 源码字号、等宽字体
│   └── PDF 选项（纸张/边距/背景色，在 PDF 弹窗配置）
│
└── 数据存储
    ├── uTools dbStorage（生产）
    ├── localStorage / IndexedDB（开发回退）
    └── 图片资产独立 key + 孤儿 GC
```

### 2.3 视图模式对比

| 模式 | 编辑引擎 | 渲染引擎 | 适用场景 | 工具栏 | 侧边栏 | 目录 |
|------|----------|----------|----------|--------|--------|------|
| **预览 (`live`)** | Milkdown WYSIWYG | Milkdown (ProseMirror) | 所见即所得写作 | 隐藏 | 显示 | 可选 |
| **分屏 (`split`)** | CodeMirror 6 | marked + highlight.js | 源码 + 实时预览 | 显示 | 显示 | 可选 |
| **源码 (`source`)** | CodeMirror 6 | 无 | 纯 Markdown 编辑 | 显示 | 显示 | 可选 |
| **专注 (`focus`)** | Milkdown WYSIWYG | Milkdown (ProseMirror) | 无干扰全屏写作 | 隐藏 | 隐藏 | 隐藏 |

> 大文件（>200KB）自动切换到分屏模式以提升 WYSIWYG 渲染性能。

## 3. 用户流程

### 3.1 核心操作流：新建笔记 → 编辑 → 保存 → 导出

```
uTools 唤起 → 输入 "md"
   │
   ├── 首次使用 → 自动创建欢迎笔记
   │
   ├── 新建笔记（+ 新建）
   │    └── 自动从第一个标题提取笔记名
   │
   ├── 选择视图模式（预览 / 分屏 / 源码 / 专注）
   │
   ├── 编辑（可粘贴图片、使用格式化工具栏）
   │
   ├── 内容变化 → 防抖 300ms → 自动持久化
   │
   ├── 导出 .md → ⬇ → 选择路径
   ├── 导出 PDF → PDF 按钮 → 配置纸张/边距 → 保存
   │
   └── 切换笔记 / 关闭插件 → 自动保存
```

### 3.2 文件夹管理流

```
点击侧边栏 "+" → 输入文件夹名 → Enter 创建
   │
   ├── 点击文件夹 → 过滤该文件夹下笔记
   ├── 双击文件夹 → 重命名
   ├── 删除文件夹 → 笔记移回根目录
   └── 点击 "全部笔记" → 清除过滤
```

### 3.3 搜索流

```
侧边栏搜索框输入关键词
   │
   ├── 匹配笔记标题
   ├── 匹配笔记正文（内存索引，保存时同步）
   └── 可与文件夹过滤叠加（标签搜索 → v1.1）
```

## 4. 非功能需求

### 4.1 性能指标

| 指标 | 目标 | 当前实现 |
|------|------|----------|
| 应用启动时间 | < 1s | Vite 开发模式 HMR |
| 内容输入响应 | 无感知延迟 | 预览防抖 150ms，WYSIWYG 实时渲染 |
| 持久化延迟 | < 500ms | 防抖 300ms |
| 大文件阈值 | > 200KB 切换模式 | `LARGE_FILE_THRESHOLD = 200_000` |
| 目录渲染 | 虚拟列表 > 150 条时启用 | `VIRTUAL_THRESHOLD = 150` |
| 单张图片上限 | ≤ 2MB，长边 ≤ 1920px | 压缩后入库 |
| 存储占用 | dbStorage / localStorage 配额 | 配额异常有通知 |

### 4.2 兼容性

| 环境 | 支持情况 |
|------|----------|
| uTools（Windows / macOS） | ✅ 主目标平台 |
| 浏览器（开发调试） | ✅ localStorage + IndexedDB 回退 |
| Node.js | 仅构建时依赖 |

### 4.3 安全性

| 方面 | 策略 |
|------|------|
| 数据存储 | 全部本地，不上传任何数据 |
| 文件操作 | uTools 对话框，用户主动选路径 |
| XSS | marked 输出转义；剥离危险 HTML |
| 存储异常 | 捕获异常并通知（空间不足 / 保存失败）|

## 5. 数据模型

### 5.1 笔记 (Note)

```typescript
interface Note {
  id: string
  title: string        // 自动从内容首个标题提取
  content: string      // Markdown 正文（可含 markflow-asset:// 引用）
  folderId?: string
  tags: string[]       // 预留，UI 待 v1.1
  createdAt: number
  updatedAt: number
}
```

### 5.2 笔记列表项 (NoteListItem)

```typescript
interface NoteListItem {
  id: string
  title: string
  folderId?: string
  updatedAt: number
}
```

### 5.3 文件夹 (Folder)

```typescript
interface Folder {
  id: string
  name: string
  order: number
}
```

### 5.4 应用设置 (AppSettings)

```typescript
interface AppSettings {
  theme: 'light' | 'dark' | 'system'
  fontSize: number
  editorFontFamily: string
  previewVisible: boolean
  sidebarVisible: boolean
  pdfExport?: {
    pageSize: 'A4' | 'A3' | 'Letter'
    margin: 'default' | 'narrow' | 'wide' | 'none'
    printBackground: boolean
  }
}
```

### 5.5 存储 Key 设计

| Key | 内容 | 序列化 |
|-----|------|--------|
| `markflow_note_list` | 笔记列表 `NoteListItem[]` | JSON |
| `markflow_note_{id}` | 单篇笔记 `Note` | JSON |
| `markflow_folder_list` | 文件夹列表 `Folder[]` | JSON |
| `markflow_settings` | 应用设置 `AppSettings` | JSON |
| `markflow_asset_index` | 图片索引 `AssetIndexItem[]` | JSON |
| `markflow_asset_{id}` | 图片记录 `AssetRecord` | JSON（含 base64） |

## 6. 竞品分析

### 对比维度

| 维度 | MarkFlow | Typora | VS Code + 插件 | Obsidian | Notion |
|------|----------|--------|----------------|----------|--------|
| 启动速度 | ⚡ 极快（uTools） | 较快 | 较慢（IDE） | 中等 | 慢（Web）|
| 离线能力 | ✅ 全离线 | ✅ 全离线 | ✅ 全离线 | ✅ 全离线 | ❌ 需网络 |
| 所见即所得 | ✅ Milkdown | ✅ 专有引擎 | ❌ 源码为主 | ❌ 源码为主 | ✅ |
| 多笔记管理 | ✅ 文件夹 | ❌ 单文件 | ✅ 文件夹 | ✅ 双链 | ✅ 数据库 |
| 搜索 | 🔄 标题+正文 | ❌ 无 | ✅ 全文 | ✅ 全文 | ✅ 全文 |
| 导出格式 | 📄 .md / PDF | 📄 .md / HTML / PDF | 📄 .md | 📄 .md | 📄 受限 |
| 图片 | ✅ 粘贴入库 | ✅ | ✅ | ✅ | ✅ |
| 标签系统 | ❌ 未接入 | ❌ | ❌ | ✅ | ✅ |
| 云同步 | ❌ | ❌ | ❌ | 付费 | ✅ 内置 |
| 价格 | 🆓 免费 | 💰 $14.99 | 🆓 免费 | 💰 同步付费 | 🆓 个人免费 |

### 差异化定位

MarkFlow 的核心竞争力在于 **「uTools 插件生态内的极速 Markdown 写作体验」**：

1. **零切换成本** — uTools 快捷键 + 回车即可开始写作
2. **本地优先** — 笔记与图片均存本地，隐私安全
3. **双引擎兼容** — WYSIWYG 直觉写作 + 源码精确控制
4. **uTools 原生导出** — 一键 PDF（Chromium printToPDF）

## 7. 已知限制与技术债务

| 编号 | 问题 | 影响 | 计划 |
|------|------|------|------|
| TD-1 | `tags` 无 UI | 无法按标签组织 | v1.1 |
| TD-2 | 搜索缺摘要/标签/防抖 | 检索体验不完整 | v1.1 |
| TD-3 | dbStorage 无备份方案 | 配额满或误删难恢复 | v1.1 |
| TD-4 | 无排序/置顶 | 列表仅按更新时间 | v1.1 |
| TD-5 | 无自动备份/版本历史 | 误删无法找回 | v1.2 |
| TD-6 | 专注模式无格式化入口 | 需切视图或快捷键 | v1.2 |
