## 1. 产品概述

### 1.1 产品定位

**MarkFlow** 是一款基于 uTools 平台的本地 Markdown 编辑器插件，随叫随到，专注写作体验。面向需要高频记录、写作的技术人员与知识工作者。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| 快捷 | uTools 一键唤起，无需切换窗口，即输即存 |
| 本地 | 所有数据存储在本地，无需网络，隐私安全 |
| 双模式 | 所见即所得（Milkdown）+ 源码编辑（CodeMirror）双引擎 |
| 四视图 | 预览、分屏、源码、专注，适配不同写作场景 |
| 轻量 | 插件化形态，随 uTools 启动，不额外占用系统资源 |

### 1.3 用户画像

| 用户类型 | 典型场景 | 核心需求 |
|----------|----------|----------|
| 开发者 | 写技术文档、API 笔记、代码片段 | 源码编辑、代码高亮、GFM 语法 |
| 知识工作者 | 日常记录、会议纪要、知识整理 | 快捷记录、文件夹组织、全文搜索 |
| 写作者 | 长文写作、博客草稿 | 专注模式、所见即所得、导出 |

### 2.3 视图模式对比

| 模式 | 编辑引擎 | 渲染引擎 | 适用场景 | 工具栏 | 侧边栏 | 目录 |
|------|----------|----------|----------|--------|--------|------|
| **预览 (`live`)** | Milkdown WYSIWYG | Milkdown (ProseMirror) | 所见即所得写作 | 隐藏 | 显示 | 可选 |
| **分屏 (`split`)** | CodeMirror 6 | marked + highlight.js | 源码 + 实时预览 | 显示 | 显示 | 可选 |
| **源码 (`source`)** | CodeMirror 6 | 无 | 纯 Markdown 编辑 | 显示 | 显示 | 可选 |
| **专注 (`focus`)** | Milkdown WYSIWYG | Milkdown (ProseMirror) | 无干扰全屏写作 | 隐藏 | 隐藏 | 隐藏 |

> 大文件（>200KB）自动切换到分屏模式以提升 WYSIWYG 渲染性能。

## 3. 用户流程

### 3.1 核心操作流：新建笔记 → 编辑 → 保存 → 导出

```
uTools 唤起 → 输入 "md"
   │
   ├── 首次使用 → 自动创建欢迎笔记
   │
   ├── 新建笔记（点击 + 新建）
   │    └── 自动从第一个标题提取笔记名
   │
   ├── 选择视图模式
   │    ├── 预览 → 直接 WYSIWYG 编辑
   │    ├── 分屏 → 左侧源码，右侧预览
   │    ├── 源码 → 纯文本编辑
   │    └── 专注 → 全屏写作，Esc 退出
   │
   ├── 内容变化 → 防抖 300ms → 自动持久化
   │
   ├── 导出 → 点击 ⬇ → 选择路径 → 保存 .md
   │
   └── uTools 关闭 / 切换笔记 → 自动保存
```

### 3.2 文件夹管理流

```
点击侧边栏 "+" → 输入文件夹名 → Enter 创建
   │
   ├── 点击文件夹 → 过滤该文件夹下笔记
   ├── 双击文件夹 → 重命名
   ├── 右键文件夹 → 删除（确认弹窗）
   └── 点击 "全部笔记" → 清除过滤
```

## 4. 非功能需求

### 4.1 性能指标

| 指标 | 目标 | 当前实现 |
|------|------|----------|
| 应用启动时间 | < 1s | Vite 开发模式 HMR |
| 内容输入响应 | 无感知延迟 | 预览防抖 150ms，WYSIWYG 实时渲染 |
| 持久化延迟 | < 500ms | 防抖 300ms |
| 大文件阈值 | > 200KB 切换模式 | `LARGE_FILE_THRESHOLD = 200_000` |
| 目录渲染 | 虚拟列表 > 150 条时启用 | `VIRTUAL_THRESHOLD = 150` |
| 存储占用量 | 受限于 uTools dbStorage / localStorage 配额 | 有配额错误提示 |

### 4.2 兼容性

| 环境 | 支持情况 |
|------|----------|
| uTools（Windows / macOS） | ✅ 主目标平台 |
| 浏览器（开发调试） | ✅ 通过 localStorage 回退 + preload 桥接 |
| Node.js | 仅构建时依赖 |

### 4.3 安全性

| 方面 | 策略 |
|------|------|
| 数据存储 | 全部本地，不上传任何数据 |
| 文件操作 | 通过 uTools 对话框（用户主动选择路径）|
| XSS | Milkdown / marked 输出已做转义处理 |
| 存储异常 | 捕获异常并通知用户（空间不足 / 保存失败）|

## 5. 数据模型

### 5.1 笔记 (Note)

```typescript
interface Note {
  id: string           // 唯一标识 (Date.now().toString(36) + random)
  title: string        // 标题（自动从内容首个标题提取，回退首行前30字）
  content: string      // Markdown 正文
  folderId?: string    // 所属文件夹（可选）
  tags: string[]       // 标签（预留字段，UI 暂未接入）
  createdAt: number    // 创建时间戳
  updatedAt: number    // 最后更新时间戳
}
```

### 5.2 笔记列表项 (NoteListItem)

```typescript
interface NoteListItem {
  id: string
  title: string
  folderId?: string
  updatedAt: number
}
```

### 5.3 文件夹 (Folder)

```typescript
interface Folder {
  id: string
  name: string
  order: number       // 排序序号
}
```

### 5.4 应用设置 (AppSettings)

```typescript
interface AppSettings {
  theme: 'light' | 'dark' | 'system'
  fontSize: number
  editorFontFamily: string
  previewVisible: boolean
  sidebarVisible: boolean
}
```

### 5.5 存储 Key 设计

| Key | 内容 | 序列化 |
|-----|------|--------|
| `markflow_note_list` | 笔记列表 `NoteListItem[]` | JSON |
| `markflow_note_{id}` | 单篇笔记 `Note` | JSON |
| `markflow_folder_list` | 文件夹列表 `Folder[]` | JSON |
| `markflow_settings` | 应用设置 `AppSettings` | JSON |

## 6. 竞品分析

### 对比维度

| 维度 | MarkFlow | Typora | VS Code + 插件 | Obsidian | Notion |
|------|----------|--------|----------------|----------|--------|
| 启动速度 | ⚡ 极快（uTools） | 较快 | 较慢（IDE） | 中等 | 慢（Web）|
| 离线能力 | ✅ 全离线 | ✅ 全离线 | ✅ 全离线 | ✅ 全离线 | ❌ 需网络 |
| 所见即所得 | ✅ Milkdown | ✅ 专有引擎 | ❌ 源码为主 | ❌ 源码为主 | ✅ |
| 多笔记管理 | ✅ 基础（文件夹） | ❌ 单文件 | ✅ 文件夹 | ✅ 双链 | ✅ 数据库 |
| 搜索 | ✅ 标题搜索 | ❌ 无 | ✅ 全文 | ✅ 全文 | ✅ 全文 |
| 导出格式 | 📄 .md | 📄 .md / HTML / PDF | 📄 .md | 📄 .md | 📄 受限 |
| 标签系统 | ❌ 未接入 | ❌ | ❌ | ✅ | ✅ |
| 云同步 | ❌ | ❌ | ❌ | 付费 | ✅ 内置 |
| 价格 | 🆓 免费 | 💰 $14.99 | 🆓 免费 | 💰 同步付费 | 🆓 个人免费 |

### 差异化定位

MarkFlow 的核心竞争力在于 **"uTools 插件生态内的极速 Markdown 写作体验"** —— 不追求大而全，而是聚焦于：

1. **零切换成本** — 从 uTools 呼出到开始写作只需 1 次快捷键 + 1 次回车
2. **本地优先** — 所有数据在本地，隐私安全，离线可用
3. **双模式兼容** — 既满足 WYSIWYG 直觉写作，又保留源码编辑的精确控制
